Key Features
🔐
mTLS Everywhere
Mutual TLS between all services — certificate rotation automated via SPIFFE/SPIRE
📜
OPA Policy Engine
Write security policies in Rego, enforce them at admission control and runtime
🪪
SPIFFE Identity
Cryptographic service identities — no passwords, no static credentials
🚪
Dynamic Authorization
Real-time policy evaluation on every request, not just at login
🔍
Access Audit Logs
Immutable audit trail of every authorization decision with full context
🌐
Multi-cluster Support
Consistent policy enforcement across AWS, GCP, Azure, and on-prem clusters
Who Is It For?
Platform Security Teams
Lateral movement risk after a single compromised service
Every service authenticates cryptographically — compromise is contained to one identity
Compliance Officers
Proving least-privilege access for SOC 2 / ISO 27001 audits
Policy-as-code with automated audit reports showing exactly what each service can access
Cloud Architects
Inconsistent security posture across multi-cloud deployments
Single control plane enforcing identical policies on every cluster regardless of provider
Quick Start
$ git clone https://github.com/charlesbulabula/zero-trust-framework
$ cd zero-trust-framework
$ kubectl apply -f manifests/spire/          # deploy SPIFFE runtime
$ kubectl apply -f manifests/opa-gatekeeper/ # deploy policy engine
$ kubectl apply -f policies/baseline.yaml
